# OpenVLA GRPO Implementation Guide

This guide documents the current OpenVLA PPO/GRPO training pipeline in this repo.

## Scope

The trainer uses exactly two stages:

1. `warmup`
- Rollout behavior: execute actions from the L1 regression head.
- Optimization: behavior cloning (cross-entropy) on tokenized action outputs.
- Goal: bring tokenized policy to a usable success baseline.

2. `rl`
- Rollout behavior: execute tokenized policy actions only.
- Optimization: PPO/GRPO objective with clipped importance ratios.
- Goal: improve task success while preserving exploration via entropy bonus.

There is no mixed-policy stage.

## Stage Machine

The stage is selected once per rollout+update cycle and held fixed for that cycle.

- Stage selector: `_determine_training_stage()` in `OpenVLA_PPO.py`
- Cycle lock: `collect_rollouts(..., stage=stage)` and `update_policy(..., stage=stage)`
- Runtime values: `warmup` or `rl`

This lock prevents mixed-policy samples inside one PPO update.

## Warmup Exit Gate

Warmup exits when all required gate checks are satisfied:

- Minimum warmup steps reached: `l1_warmup_steps`
- Rolling tokenized validation success >= `l1_warmup_success_threshold`
- Gate holds for `l1_warmup_required_consecutive` validations

Additional safety checks:

- Hard warmup cap: `l1_warmup_max_steps` (<=0 disables)
- Anti-overclone guard: `l1_warmup_overclone_threshold` (set >1.0 to disable)

Recommended defaults in this repo:

- `l1_warmup_steps=25000`
- `l1_warmup_success_threshold=0.40`
- `l1_warmup_success_window=3`
- `l1_warmup_required_consecutive=2`
- `l1_warmup_max_steps=32000`
- `l1_warmup_overclone_threshold=0.85`

## Rollout Behavior by Stage

### Warmup

- Action source: L1 head
- Buffer stores:
  - token responses/log-probs from model forward pass
  - optional L1 action chunk targets
  - sparse episode reward (success=1, fail=0)

### RL

- Action source: tokenized head
- Buffer stores:
  - executed tokenized actions
  - old tokenized log-probs for PPO ratio
  - sparse rewards for GRPO advantage computation

## Losses

### Warmup Loss (Behavior Cloning)

Cross-entropy trains action token logits against discretized L1 action targets.

- Implementation: `_bc_update_from_l1()`
- Key stats: `train/bc_loss`, `train/bc_accuracy`, `train/bc_entropy`

### RL Loss (PPO/GRPO)

PPO clipped surrogate is used with GRPO-style advantages computed from sparse outcomes.

- Ratio: `exp(new_log_prob - old_log_prob)`
- Clipping: asymmetric bounds from `clip_ratio_low` and `clip_ratio_high`
- Entropy bonus: controlled by `entropy_coef`
- Optional KL regularization: `kl_coef`

Key stats include:

- `train/policy_loss`
- `train/clipfrac`
- `train/approx_kl`
- `train/entropy`

## Checkpoint Policy

Checkpoint policy is stage-aware and avoids interval spam.

- Latest checkpoint per stage:
  - `latest_model_stage_warmup.pt`
  - `latest_model_stage_rl.pt`
- Best checkpoint per stage:
  - `best_model_stage_warmup.pt`
  - `best_model_stage_rl.pt`

Stage metadata includes:

- current stage
- stage-local and global step
- stage-change reason
- warmup gate state

## Canonical Training Entry

Recommended launcher:

```bash
./start_ppo_training.sh \
  --task-suite libero_spatial \
  --task-id 0 \
  --timesteps 100000 \
  --seed 0
```

Direct script form:

```bash
python OpenVLA_PPO.py \
  --task-suite libero_spatial \
  --task-id 0 \
  --timesteps 100000 \
  --seed 0 \
  --warmup-min-steps 25000 \
  --warmup-max-steps 32000 \
  --warmup-success-threshold 0.40 \
  --warmup-success-window 3 \
  --warmup-success-consecutive 2 \
  --warmup-overclone-threshold 0.85
```

## Common Failure Patterns

1. Warmup never exits
- Lower threshold or required consecutive checks.
- Verify tokenized validation is actually running at `val_interval`.

2. Early RL instability
- Lower actor LR.
- Tighten clip ratios.
- Increase entropy bonus only if policy collapses too quickly.

3. CUDA OOM
- Lower batch size.
- Disable data parallel.
- Confirm no stale processes on GPU.

4. Checkpoint source mismatch
- Confirm whether training uses local checkpoint path or HF repo ID.
- Keep source fixed for reproducibility.

5. Dataset/environment mismatch
- Follow exact setup in `libero_rl/README.md`.

## Notes for Public Artifact Claims

- Label metrics as `computed` only when generated by code in this repo.
